# Требования к PNG-оверлеям рамок карточек

Документ для дизайнера или для самостоятельной нарезки ассетов под текущую вёрстку (`CardTile` / классы `wb-tcg--*`).

---

## Назначение

PNG накладываются **поверх** контента карточки (иллюстрация, заголовок, текст, тег, ссылка) или используются как **`border-image`** / **9-slice**, чтобы заменить нарисованные в CSS рамки по редкости: **common, rare, epic, legendary, mythic**.

Важно: только **оригинальная** графика / собственные формы — не копировать рамки Hearthstone, Marvel и др.

---

## Набор файлов

Один визуальный вариант на каждую редкость (итого **5** логических рамок).

Рекомендуемые имена (после экспорта положить, например, в `public/frames/`):

| Редкость   | Файл @1x (опционально) | Файл @2x (обязательно для чёткости) |
|------------|------------------------|-------------------------------------|
| Обычная    | `frame-common.png`     | `frame-common@2x.png`               |
| Редкая     | `frame-rare.png`       | `frame-rare@2x.png`                 |
| Эпическая  | `frame-epic.png`       | `frame-epic@2x.png`                 |
| Легендарная| `frame-legendary.png`  | `frame-legendary@2x.png`           |
| Мифическая | `frame-mythic.png`     | `frame-mythic@2x.png`               |

Достаточно **только @2x** (удвоенный размер), в коде масштабировать до логических px — так проще не плодить дубликаты.

---

## Два допустимых формата ассета

### Вариант 1 — «Рамка с прозрачным окном» (overlay)

Один PNG на редкость:

- **Внешний размер** совпадает с прямоугольником всей карточки (с закруглениями, как в макете).
- **Центральная область** (под арт + текст + ссылку) — **полностью прозрачная** (`alpha = 0`), без полупрозрачной «плёнки» поверх будущего текста.
- Все блики, тени и «камень» по референсу могут быть **в непрозрачных/полупрозрачных частях только на самой рамке** (периметр), не заходя на контент.
- **Безопасная зона контента**: от **внутреннего края** декоративной рамки до начала иллюстрации — минимум **8–12 px @1x** (16–24 @2x), чтобы текст не наезжал на орнамент.

Учитывайте, что в UI уже есть:

- круглый бейдж **W** (левый верх);
- плашка **редкости** (правый верх);
- ссылка **«Читать статью»** внизу.

Либо **временно скрываем** их в коде под ваш полноэкранный макет, либо **вписываете** вырез/орнамент так, чтобы не пересекаться — это нужно согласовать при внедрении.

### Вариант 2 — 9-slice (гибко по высоте)

Три столбца × три ряга (углы + ребра + центр). Центральная клетка обычно прозрачная или однотонная подложка.

- Экспорт: одна PNG **полоса** или готовый **9-patch** (Android) / отдельные 9 файлов — как удобнее фронту.
- **Края** (толщина «разреза» для slice): зафиксировать в px и передать вместе с файлами (например слева/справа **24 @2x**, сверху/снизу **32 @2x**).

Подходит, если высота карточки сильно плавает из‑за длины описания.

## Размеры и плотность

Ориентиры под текущий UI (можно скорректировать после первого импорта):

| Контекст        | Логическая ширина (@1x) | Рекомендуемая ширина холста @2x |
|-----------------|-------------------------|----------------------------------|
| Сетка 2 колонки | ~260 px                 | **520 px**                       |
| Lightbox (zoom) | ~360 px                 | **720 px**                       |

**Соотношение сторон** холста лучше зафиксировать одним для всех редкостей, например **3 : 4** или **5 : 7** (как в вашем референсе) — тогда в CSS задаётся `aspect-ratio`, и карточки не «прыгают».

## Технические требования к файлам

- **Цветовое пространство**: **sRGB**.
- **Прозрачность**: канал **alpha** обязателен там, где должен просвечивать контент.
- **Битая подложка**: без белого фона под всей картой (если только это не часть дизайна рамки).
- **Резкость**: без лишнего JPEG; предпочтительно **PNG** или **WebP** с альфой.
- **Вес**: по возможности сжимать (TinyPNG и т.п.), цель — до **~200–400 KB** на @2x-файл или меньше.
- **Мифическая** рамка: если задуман **шиммер**, можно отдать **статичный** PNG (перелив оставить в CSS) или короткий **спрайт / несколько кадров** — тогда отдельное ТЗ на анимацию.

---

## Слои и свечение

- **Внешнее свечение** (glow): либо **входит в размер холста** с полупрозрачными пикселями по краям, либо задаётся отдельным слоем и документируется: «glow выступает на **N px** за габарит карточки» — чтобы в CSS добавить `padding`/`margin` и не обрезать ореол `overflow: hidden`.
- **Внутренний блик** на рамке — допустим; на область текста — нет.

---

## Что передать разработчику вместе с PNG

1. Имена файлов и папка (`public/frames/…`).
2. Выбранный вариант: **overlay с окном** или **9-slice** + **координаты slice** (px @2x).
3. Точное **соотношение сторон** и **логическая ширина** @1x.
4. Отступ **safe area** для контента от внутренней кромки рамки (@1x и @2x).
5. Нужно ли **скрывать** встроенные бейджи W / редкость / камень под полноценный макет.

После этого в коде добавятся `background-image` / слой `<img class="wb-tcg__frame-overlay">` и при необходимости отключится часть CSS-рамок у `wb-tcg--*`.
